# Insurance Fraud Detection System

## Problem

Fraud in insurance claims leads to significant financial losses, and identifying suspicious patterns manually is inefficient and error-prone.

This project focuses on building a data-driven system to detect potential fraud by analyzing structured claim data and identifying anomalies.

---

## Objective

To design a system that can:

- Process and clean insurance claim data  
- Identify irregular patterns in claims  
- Support fraud detection through data analysis  
- Provide insights via dashboards for decision-making  

---

## Approach

The system follows a structured pipeline:

1. Data ingestion and preprocessing  
2. Data cleaning and validation  
3. Feature analysis and anomaly detection  
4. Output generation and visualization  

---

## Key Design Decisions

### 1. Data Validation Layer
Ensured data consistency by handling missing values and incorrect entries before analysis.

---

### 2. Anomaly Detection Approach
Used rule-based and statistical techniques to identify unusual claim patterns instead of relying only on model accuracy.

---

### 3. Dashboard for Decision Support
Built a Power BI dashboard to:

- Monitor fraud-related patterns  
- Analyze claim trends  
- Support investigation workflows  

---

## Key Insights

- Certain claim patterns show abnormal behavior compared to normal distributions  
- Repeated anomalies in specific segments indicate higher fraud risk  
- Data quality significantly impacts fraud detection reliability
